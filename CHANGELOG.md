# RedBox 更新日志

## v2.4.0 (2026-06-20)

### 浏览器控制与真实 Chrome 验收
- 新增 Codex-style browser facade，桌面端通过 `Operate(resource="browser", operation="control")` 统一暴露打开、接管、导航、DOM 读取、locator 查询、点击、输入、滚动、截图和标签释放能力。
- 浏览器插件新增 `scripts/browser-client.mjs`，支持按 `setupBrowserRuntime -> agent.browsers.get("extension")` 的对象 API 调用 RedBox browser-control 后端。
- native host 安装器支持自动发现 Chrome / Edge / Brave 扩展，生成使用绝对 Node 路径的 launcher，避免 GUI Chrome PATH 不一致导致 native host 无法启动。
- 新增真实链路诊断和回归脚本：`pnpm diagnose:browser-control` 检查 manifest、launcher、socket 和扩展转发；`pnpm smoke:browser-control` 验证 native socket、tools/list、标签创建和 DOM 读取。

### 用户可见控制标识与隐私
- 被 AI 接管或创建的标签页会显示 `RedBox 控制中` 页面内标签，释放、finalize 或 turn 结束后自动移除。
- 浏览器上下文、历史搜索和剪贴板读取不再作为无感只读能力暴露，默认进入更高风险授权路径。
- 文档明确区分“网页读取”和“真实浏览器操作”，真实验收必须使用安装了 RedBox 插件的 Google Chrome，而不是只跑 Chromium smoke。
- macOS Keychain / Chrome Safe Storage 授权不应成为浏览器控制前置条件，测试时遇到此类系统隐私提示应拒绝并修复 launcher/profile 路径。

### AI 工具面与桌面端稳定性
- AI tool surface 向 Codex 风格收敛，命令执行、浏览器控制、计划工具和兼容层边界更明确。
- 改进运行过程说明回放，恢复会话后能更稳定地复盘工具调用和过程事件。
- 修复桌面端启动后没有恢复上次应用视图的问题。

### 版本与分发
- 桌面端版本号更新到 `2.4.0`。
- 浏览器插件版本号更新到 `2.4.0`。
- 本次 `v2.4.0` 作为 GitHub prerelease 发布，已提供 macOS、Windows 多架构安装包和浏览器插件包。

## v2.2.1 (2026-05-30)

### 浏览器插件与知识库
- 修复微信公众号文章保存链路，长文和多图文章不会再因为请求体过大而保存失败。
- 公众号文章会保留可阅读的富文本快照，正文图片继续作为知识库素材沉淀，减少后续打开文章时的丢图概率。
- 修复链接文章、公众号文章在知识库中的类型识别和筛选映射，已保存的文章可以稳定出现在对应分类里。
- 确认通用网页链接、微信公众号文章、知乎回答和知乎专栏文章都走 app 端 Knowledge API 的正式入库类型。

### 工作台与运行稳定性
- 自由创作入口改为默认工作区视图，进入 app 后更快回到图片、视频、音频等核心创作任务。
- 拆分媒体生成任务队列，减少不同创作模式之间的任务状态串扰。
- 强化媒体任务超时处理和团队成员唤醒运行时，降低长任务、协作任务卡住或状态不同步的概率。
- 修复官方微信登录确认后 session 未规范化导致无法进入工作台的问题。
- 补齐 TTS 模型能力识别，降低音色、语音模型选择和真实可调用能力不一致的问题。

- 重点增强东南亚、欧洲和跨境电商平台的多国家、多语言详情页支持，平台版本会自动带入目标市场和语言要求

- 支持的详情页语言包括：英语、德语、法语、西班牙语、马来语、泰语、繁体中文、印尼语、越南语、葡萄牙语、意大利语、日语、阿拉伯语、荷兰语、波兰语、瑞典语、丹麦语、芬兰语、挪威语、捷克语、斯洛伐克语、斯洛文尼亚语、克罗地亚语、匈牙利语、罗马尼亚语、爱沙尼亚语、拉脱维亚语、立陶宛语、保加利亚语、土耳其语、俄语、韩语、阿塞拜疆语、哈萨克语、亚美尼亚语、吉尔吉斯语、乌兹别克语、格鲁吉亚语、塔吉克语
- 支持从媒体库和生成结果中回收图片素材，生成后的图片可以直接进入商品详情页工作区继续编辑和复用
- 优化详情页工作区布局，商品信息、平台版本、素材选择和生成结果在同一创作路径中完成

### 生成工作台与媒体任务
- 生成工作台拆分出更稳定的请求校验、提交 payload、Agent 上下文和数字人音频解析模块
- Agent 模式现在会带上更明确的当前请求、最近资产、可用音色和模糊引用策略，减少“上一张图”“刚才的音频”等上下文丢失
- 媒体任务支持归档删除，生成结果从工作台移除后不再依赖页面本地队列兜底
- 生成历史补齐 `video_sequence` 等任务类型识别，视频序列类结果可以更稳定地回到生成工作台
- 视频生成时长范围扩展到 1-15 秒，短片段和稍长片段都能走统一任务链路

### 官方 AI 与模型配置
- 修复官方 AI 默认模型缺失后的自动补齐逻辑，启动、设置保存和官方登录同步后都会尝试修复空缺路由
- 专用能力不再误用聊天模型兜底，图片、视频、Embedding、转写、TTS、音色克隆等能力必须使用对应模型配置
- 官方模型同步继续保留用户已设置的自定义路由，只补齐缺失项，避免升级或刷新后把用户选择覆盖回默认模型
- 官方调用记录排序和分页更稳定，最近请求、模型和用量记录展示更可靠

### 数字人与视频处理
- VideoRetalk 参考视频预处理拆分为独立模块，继续支持本地视频探测、尺寸规范化和上传前转码
- 数字人流程的音频生成、参考视频准备和视频提交链路进一步收口，失败原因更容易定位
- 媒体运行时补齐任务归档、并发槽位和任务类型结构，长时间视频任务与多媒体任务并行时更稳定

### 浏览器插件
- 插件增加微信公众号文章域名权限，后续采集公众号内容时无需重新安装扩展权限包

## v2.1.0 (2026-05-20)

### 数字人视频
- 新增数字人视频生成链路，支持使用角色参考视频和配音音频生成口播视频
- 角色库支持保存参考视频，数字人创作可直接复用角色的视频、声音和基础设定
- 数字人生成接入媒体任务系统，支持提交、等待、解析成片地址和回到工作台查看结果
- 数字人入口并入生成工作台，减少角色、音频、视频结果之间来回切换
- 支持基于角色视频进行音色克隆，让数字人口播更接近角色本身的声音表现

### 音频与音色
- 音频生成支持按语言筛选音色，降低跨语言选错音色导致的生成质量问题
- 音色列表跟随 TTS 模型变化，不同模型只展示对应可用音色
- 修复模型绑定的声音克隆槽位，角色的多模型音色配置更稳定
- 修复 TTS 模型列表来源，避免页面选项和真实可调用能力不一致

### 知识库与采集
- 浏览器插件新增知乎内容采集，知乎页面内容可以进入 RedBox 知识库并参与后续检索
- 修复知乎采集路由和索引路径，提升采集后入库与检索稳定性
- 优化知识库视频转写重试，减少临时上游失败导致的视频资料入库失败
- 知识库空状态增加插件入口提示，首次配置外部采集更直接

### 媒体与聊天
- 聊天输入框支持直接粘贴图片，并作为附件进入当前对话和 Agent 工具调用
- 媒体结果增加保存相关操作，生成后的图片、音频、视频更容易沉淀复用
- 支持更多 Windows 本地媒体路径，减少本地媒体素材预览或生成时的路径访问失败

### 拖拽文件快捷功能
- 支持将本地文件直接拖入聊天窗口，快速作为附件加入当前对话
- 拖入图片、视频等媒体文件后，会自动进入后续预览、分析和 Agent 工具调用链路
- 浏览器插件支持拖拽网页图片保存到 RedBox 素材库，减少从网页素材到本地素材库的手动下载步骤

![拖拽文件快捷功能演示](https://github.com/Jamailar/RedBox/releases/download/v2.1.0/redbox-2.1.0-drag-file-shortcut-demo.gif)

### 修复
- 修复聊天快捷提示词被显示到正式消息里的问题
- 修复已有 RedClaw 空间无法进入新样式流程的问题
- 修复官方 AI 设置中的缓存用量显示问题
