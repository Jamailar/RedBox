# 一、产品概述

## 1. 产品定位

本产品是一款**本地优先（local-first）的 AI 内容创作工作台**，面向以小红书为代表的内容创作者，通过「内容采集 → 结构分析 → 灵感与选题 → 标题与正文创作」的一体化流程，帮助用户**学习爆款结构、沉淀个人风格、提高创作效率与稳定性**。

产品强调：

* AI 作为“可解释的执行者”，而不是黑盒生成器；
* 所有过程可视化、可回放、可复用；
* 知识库与风格资产归用户本地所有。

## 2. 目标用户

* 小红书 / 短视频平台的中高频内容创作者
* 有明确账号定位与长期创作目标的人
* 不满足于“随便生成”，而希望**学习、控制与复用内容结构与风格**的用户

## 3. 核心价值

* 从“灵感不稳定”变为“结构化、可持续产出”
* 从“模仿爆款”变为“沉淀自己的爆款方法论”
* 从“单次生成”变为“可积累的内容资产系统”

---

# 二、核心使用流程（用户视角）

## 流程总览

1. 浏览器插件采集笔记内容
2. 客户端本地存储并标准化内容
3. 运行 AI 分析任务（结构 / 技巧 / 风格）
4. 沉淀为可复用的风格与知识资产
5. 基于资产进行选题、标题、正文创作
6. 所有过程与产物可回看、可对比、可再次生成

---

# 三、系统整体架构（产品级）

## 1. 三层结构

### （1）采集层：浏览器插件

* 平台：Chrome / Edge
* 功能：

  * 采集小红书笔记的标题、正文、标签、作者、发布时间、互动数据
  * 将内容发送至本地客户端（localhost）或云端中转（可选）
* 不做分析、不做 AI 调用

### （2）工作台层：桌面客户端

* 负责：

  * 内容资产管理
  * 知识库管理
  * 任务调度与可视化
  * AI 执行过程展示
* UI 核心页面：

  * 内容库
  * 知识库
  * 工作台（任务执行）
  * 产物中心

### （3）AI 内核层

* 基于 LangChain / LangGraph
* 支持多模型接入（Claude / OpenAI / 本地模型）
* 以“Skill（技能）”为最小执行单元
* 所有输出必须结构化并产生事件流

# 四、内容与数据模型设计

## 1. 内容资产分层

### Inbox（原始采集）

* 插件采集的原始内容
* 原样保存，仅加采集元信息

### Canonical（标准化内容）

* 统一字段结构的“内部标准笔记对象”
* 所有 AI 分析的输入来源

### Derived（派生产物）

* 结构分析结果
* 选题卡
* 标题方案
* 正文草稿

## 2. 知识库（Knowledge Base）结构

### KB-Account（账号画像）

* 账号类型
* 创作目标（涨粉 / 转化 / 人设）
* 目标受众
* 语气与风格偏好
* 禁忌与边界

### KB-Style（风格与爆款样本）

* 收藏的爆款笔记结构
* 优秀标题样本
* 个人历史高表现内容
* 带标签与适用范围

### KB-Facts（事实与素材）

* 产品资料
* 门店/城市信息
* 可引用事实
* 要求来源与时间

# 五、AI Skills（核心功能）

所有 Skills 均遵循统一执行结构：

* 输入：任务参数 + Run Context（账号画像）+ 内容引用
* 中间：按需检索知识库（RAG）
* 输出：结构化 JSON + 文本产物 + 事件流

## Skill 1：笔记结构分析

* 拆解：主题、钩子、痛点、卖点、节奏
* 输出可视化结构卡片

## Skill 2：笔记创作（正文 + 标题）

* 支持参数化创作（字数、语气、转化强度）

## Skill 3：视频文字稿结构分析

* 钩子 / 观点 / 反转 / CTA 拆解

## Skill 4：构思选题

* 生成结构化选题卡

## Skill 5：选题打分

* 多维评分 + 可解释原因

## Skill 6：多标题方案生成

* 多策略标题 + 标签说明

# 六、任务系统与可视化

## 1. 任务执行模型

* 任务队列（支持并发限制）
* 每个任务产生事件流：

  * step_start
  * step_done
  * artifact_saved
  * warning / error

## 2. UI 展示原则

* 默认展示结构化进度
* 原始模型输出可折叠查看
* 所有产物可点击定位

---

# 七、客户端核心页面

## 1. 内容库

* Inbox / Canonical / Derived 分视图
* 状态标签（未分析 / 已分析 / 已沉淀）

## 2. 知识库

* 账号画像编辑
* 风格样本管理
* 事实素材管理

## 3. 工作台

* 选择 Skill
* 选择输入内容
* 实时查看执行过程

## 4. 产物中心

* 标题库
* 正文稿库
* 选题池
* 支持对比与导出

---

# 八、非功能性要求

* 本地优先，离线可用（除模型调用）
* 模型可替换、成本可控
* 数据可导出、可迁移
* 桌面端跨平台发布

---

# 九、产品阶段规划（建议）

## Phase 1（MVP）

* 插件采集 + 本地内容库
* 笔记结构分析
* 多标题生成

## Phase 2

* 账号画像 + 风格知识库
* 选题构思与打分

## Phase 3

* 正文创作流水线
* 历史表现反馈与风格强化

# 十、产品原则（不可妥协）

* 不做黑盒生成
* 不丢用户资产
* 不强绑定单一模型
* 不牺牲可解释性换“看起来很聪明”
