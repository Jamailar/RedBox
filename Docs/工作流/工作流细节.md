AI Agent 思维链 (Chain of Thought) 实现详解
在 AI Agent（智能体）的工程实现中，“思维链”（Chain of Thought, CoT）不仅仅是 Prompt 技巧，而是由“系统提示词 + 状态管理 + 循环执行机制”共同构成的一套运行逻辑。

它将一个复杂任务拆解为：感知 (Observation) → 思考 (Thought) → 决策 (Action) → 执行 (Execution) → 观察结果 (Next Observation) 的闭环。

一、思维链的核心构成
一个完整的工程化思维链由以下四个部分组成：

1. 结构化系统提示词 (Structured System Prompt)
这是思维链的“宪法”。必须显式规定模型如何“思考”。

角色定义：你是谁（如“资深工程师”）。
思考协议：强制模型在调用工具前，必须在一个 <thought> 标签或特定 JSON 字段中写下推理过程。
工具描述：准确描述可用工具的能力和参数格式。
2. 对话历史与状态 (Conversation History & State)
思维链的“短期记忆”。

Messages 列表：[UserMessage, AI_Thought, AI_Call, Tool_Output, AI_Thought_2...]
线性增长：每一次循环的输入，都包含之前的所有步骤结果。
3. 循环控制器 (The Loop / Orchestrator)
这是思维链的“心脏”。它不是一次 API 调用，而是一个 Client 端的 while 循环。

判断条件：模型是否发出了 Stop 信号？任务是否完成？是否达到最大步数？
错误处理：如果工具报错，将错误信息作为 Observation 喂回给模型，让其自我修正。
4. 工具执行层 (Executor)
思维链的“手脚”。

接收模型的结构化指令（JSON）。
执行实际代码/命令。
返回标准化结果（stdout/stderr）。

三、数据结构示例
在代码层面，思维链通常表现为这样一个不断追加的列表：

第 1 步：用户请求
// Prompt 输入
[
  {"role": "user", "content": "帮我把 README.md 里的版本号改成 1.0.1"}
]
第 2 步：模型思考与行动 (LLM Output)
// Response 输出
{
  "role": "assistant",
  "content": null,
  "tool_calls": [
    {
      "function": {
        "name": "read_file",
        "arguments": "{\"path\": \"README.md\"}"
      }
    }
  ]
}
(注：有些实现会强制模型先输出一段 text 解释思考过程，再输出 tool_calls)

第 3 步：工具回填 (Tool Output)
// 下一轮 Prompt 输入（追加了前两步）
[
  {"role": "user", "content": "..."},
  {"role": "assistant", "tool_calls": [...]},
  {
    "role": "tool", 
    "name": "read_file", 
    "content": "# Project RedConvert\nVersion: 1.0.0\n..." 
  }
]
第 4 步：模型再次思考 (LLM Re-eval)
// LLM 看到文件内容后，决定修改
{
  "role": "assistant",
  "tool_calls": [
    {
      "function": {
        "name": "replace_string",
        "arguments": "{\"path\": \"README.md\", \"old\": \"1.0.0\", \"new\": \"1.0.1\"}"
      }
    }
  ]
}
四、核心实现代码伪代码 (Pythonic)
history = [system_prompt, user_message]
while True:
    # 1. 思考 (Thought)
    response = llm.chat(messages=history)
    history.append(response.message)
    
    # 2. 判断是否结束
    if not response.tool_calls:
        print("Final Answer:", response.content)
        break
        
    # 3. 执行工具 (Action)
    for tool_call in response.tool_calls:
        tool_name = tool_call.function.name
        tool_args = json.loads(tool_call.function.arguments)
        
        print(f"Executing {tool_name} with {tool_args}...")
        try:
            result = tools[tool_name](**tool_args)
        except Exception as e:
            result = str(e) # 4. 错误也是一种观察
            
        # 5. 观察 (Observation)
        history.append({
            "role": "tool",
            "tool_call_id": tool_call.id,
            "content": result
        })

五、总结
思维链的本质不是模型“顿悟”了，而是通过工程手段强制模型慢下来：

显式化：把隐式的思考步骤变成显式的 API 交互。
闭环化：把“执行结果”作为新知识喂回给大脑。
状态化：维护一个不断增长的 Context Window 作为短期记忆。
