# Image Prompt Optimizer

用于 RedBox 生图链路的内置提示词优化技能。

在聊天里准备调用 `app_cli(command="image generate ...")` 时，也应先使用它整理最终提示词。

## 目标

1. 把用户原始中文需求转成更稳定、更可执行的最终生图提示词。
2. 在保留主体和创意意图的前提下，补足构图、镜头、材质、光线、色彩、环境、风格与画面密度。
3. 针对不同生图模式控制自由度，先说清楚什么必须保持不变，再决定什么可以增强或改写。
4. 让最终提示词更像“成片要求”，而不是散乱标签堆。

## 工作方式

1. 先判断当前是文生图、参考图引导还是图生图。
2. 先提炼主体锚点和不变项，再补足场景、机位、光线、材质、氛围和完成度。
3. 优先写模型真正能看见的视觉信息，不要把后台字段、UI 术语或排版说明直接抄进画面。
4. 若用户给了比例、尺寸、质量、标题等信息，把它们转成构图或版式约束，不要机械复述。

## 输出契约

- 只输出最终提示词，不输出解释、分析、Markdown、编号或 JSON 外文本。
- 最终提示词默认使用中文主干；仅在模型常见的视觉标签更稳定时，补少量英文风格词。
- 默认写成一段可直接投喂模型的完整视觉指令；允许逗号分句，但不要退化成碎片化标签云。
- 默认补足：主体、场景、构图、镜头/视角、光线、材质、色彩、氛围、清晰度要求。
- 优先使用具体画面语言，例如镜头、景别、布光、材质、空间关系、背景密度、表情动作，而不是空泛质量词。
- 禁止凭空扩写与用户目标冲突的新剧情、新主体、新品牌、新文案。

## 模式规则

- `text-to-image`：允许补足完整视觉设定，但不要把画面写成散乱关键词堆。
- `reference-guided`：主体身份、构图重心、色彩语言、服装/材质和镜头关系优先继承参考图，再补足环境与质感要求。
- `image-to-image`：默认保持原图主体、姿态和核心轮廓，只允许围绕风格、光线、背景、细节密度做受控改写。

## 强制约束

- 用户没指定风格时，不要硬塞夸张艺术流派；优先自然、清晰、可执行。
- 除非用户明确要求画面中出现文字或 logo，否则不要让模型渲染提示词原文、说明文字、布局标签、字体/配色说明、水印、AI 标签或无意义乱码。
- 若用户明确要“封面感 / 商业海报 / 小红书”，优先强调主体突出、留白、标题区、视觉中心和可读性。
- 若用户明确要“真实摄影”，优先写镜头、焦段、景深、自然光/布光、材质细节；不要混入明显插画术语。
- 若用户明确要“插画 / 3D / 海报 / 国潮 / 二次元”，风格词要集中，不要和纪实摄影词混搭。
- 参考图引导和图生图必须优先定义“不变项”，如人物身份、面部特征、产品轮廓、主色关系、主要机位，不要先写花哨风格。
- 不要输出负向提示词字段；如果需要约束，直接以内联短句写进最终提示词。

## 典型画面锚点

- 人物图：年龄感、发型、服装、表情、姿态、机位、背景关系、皮肤与面料质感。
- 产品图：主体摆位、材质、反光控制、商业布光、背景干净度、展示角度和使用情境。
- 空间场景：前中后景、机位高度、采光逻辑、材质层次、生活痕迹和视觉动线。
- 封面/海报：主体突出、视觉中心明确、背景不过载、保留可放标题的干净区域，避免把文案直接画成乱码。
