# Writing Style Creator

用于创建、更新和迭代个人写作风格技能。

目标不是一次性模仿某篇文章，而是把目标作者的稳定表达习惯、判断方式、结构偏好和禁区，沉淀成一个能持续演化的 style skill。

## 触发场景

以下情况默认应该使用这个技能：

1. 用户明确要求学习某种写作风格。
2. 用户明确对写作风格有要求，希望按某个人、某类作者或某种个人气质去写。
3. 用户想建立、沉淀或训练自己的个人写作风格。
4. 用户已经用某个风格技能写过稿，正在做复盘、修订或继续逼近目标风格。

如果用户只是要一篇普通内容，没有风格学习、风格约束、个人风格沉淀或复盘更新需求，不优先使用这个技能。

## 工作模式

1. 冷启动，用户给少量样本、口头描述、链接或片段，你先搭出第一版风格技能。
2. 资料库建模，用户要求一次性读取资料库、知识库或某个作者材料集，你先吃透材料，再产出初版风格技能。
3. 迭代修订，用户已经有现成风格技能或写作样本，你结合改稿、复盘和新资料继续更新。

## 核心原则

- 先建风格证据表，再下规则。
- 区分稳定特征、暂定特征和一次性噪音。
- 正向规则和反向禁区都要写。
- 风格技能必须可更新，不把第一版当成最终版。
- creator 的终点是产出可直接用于写作的风格技能，而不是停留在分析报告。

## 默认流程

1. 先判断目标作者、目标场景和当前模式。
2. 建立风格证据表，至少覆盖世界观、语气、结构、开头方式、转场、标志性表达、禁区、资料来源权重。
3. 优先从内置 `writing-style-base` 技能或 [references/blank-style-skill-template.md](references/blank-style-skill-template.md) 起步，作为目标技能骨架。
4. 如果用户要求一次性从资料库建立风格，按 [references/style-bootstrap-from-library.md](references/style-bootstrap-from-library.md) 先全量读取高信号资料，再沉淀初版规则。
5. 如果任务带改稿或复盘，按 [references/style-evolution-loop.md](references/style-evolution-loop.md) 把新增观察写回目标技能，而不是只改当前稿子。
6. 风格技能创建完成后，后续写作、改写、扩写、润色默认切换到新建的风格技能执行。

## 写作接力规则

- 当 creator 已经产出目标风格技能后，后续正文写作优先使用那个目标风格技能。
- creator 负责建模、更新和复盘，不负责长期替代目标风格技能本身。
- 当用户说按这个风格写一篇、继续用这个风格改、把这段改得更像我时，应该调用或更新对应的目标风格技能，而不是每次都重新从头建模。
- 当用户对成稿提出反馈时，先完成本次改稿，再把有效反馈写回目标风格技能，形成下一轮更稳定的写作规则。

## 输出要求

- 优先交付可复用的风格技能，不是一次性文案。
- 至少包含，触发场景、适用写作任务、核心气质、结构习惯、语言特征、绝对禁区、自检流程、迭代记录。
- 证据不足时明确标记为暂定规则，并继续向用户要样本，不要假装已经完全掌握。
