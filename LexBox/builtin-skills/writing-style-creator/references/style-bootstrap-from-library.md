# 从资料库建立风格

当用户要求一次性从资料库、知识库或作者材料集建立风格时，按这个流程执行。

## 目标

不是把所有内容总结成一个抽象人设，而是从全量材料里抽取稳定、可复用、可落到写作技能里的风格证据。

## 步骤

1. 先确定资料范围。
   如果用户已经指定作者资料夹、知识库条目或 workspace，就按指定范围读取。
   如果没有指定，但上下文里只有一个明显作者对象，可以默认从该对象的资料开始。
2. 给资料分级。
   优先级默认是，正式发布文章 > 作者自己写的提纲或长笔记 > 演讲/口述转写 > 与写作相关的聊天记录 > 他人总结这位作者的介绍。
3. 批量读取高信号资料，先做风格证据表。
   至少抽取这些维度，世界观、读者关系、开头方式、结构推进、论证习惯、常用词组、情绪表达、禁区。
4. 给每条观察标记证据强度。
   单一来源或单篇爆文里出现一次的，先标记为暂定。
   跨多份资料重复出现，或者被用户明确确认的，才能升级为稳定规则。
5. 再生成第一版目标风格技能。
   优先使用 `blank-style-skill-template.md` 的结构，把稳定规则写进核心部分，把暂定规则放进迭代记录或不确定项。

## 结果要求

- 初版技能必须说明资料范围和判断置信度。
- 不能因为看了很多资料，就假装所有细节都已经稳定。
- 如果资料横跨不同阶段、不同平台或不同人格面向，要显式写出分层，而不是硬揉成一种声音。

## 常见误判

- 把某一篇爆款文章的特殊写法当成长期风格。
- 把主题偏好误判成语言风格。
- 只学到口头禅，没学到判断方式和读者关系。
- 没有区分作者亲笔内容和别人转述内容。
