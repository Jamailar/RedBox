# 风格技能迭代规则

这个技能的关键不是第一次建模，而是后续能不能越来越像。

## 迭代入口

- 用户改稿后的最终版本
- 用户明确说喜欢或不喜欢的段落
- 新增的原作者样本
- 用户对上一次模仿结果的复盘

## 每轮迭代都做的事

1. 先对比旧规则和新证据。
2. 把变化分成四类：
   - 稳定增强，重复出现，可以升为正式规则
   - 软偏好，这次成立，但证据还不够
   - 硬禁区，用户明确否定，应该立刻写进禁区
   - 场景限定，只在特定平台、栏目或时期成立
3. 更新目标技能，而不是只改当前文章。
4. 在 `迭代记录` 里写清本次新增、删除、降级和待确认项。

## 升级规则

- 单次样本出现，不自动升级成核心规则。
- 至少跨多个样本重复，或者得到用户明确确认，才升级为稳定规则。
- 明确被用户否掉的表达，直接加入禁区，不需要等待复现。

## 降级规则

- 如果旧规则连续被新样本打脸，就降级成暂定规则。
- 如果发现某条规则只属于特定时期或平台，就改写成场景限定，而不是继续当通用规则。

## 复盘输出

每次复盘后，至少保留四类信息：

- 这次确认了什么
- 这次否掉了什么
- 这次新增了什么
- 还有什么没把握

目标是让风格技能越来越像目标作者，而不是越来越像上一次模型生成的结果。
