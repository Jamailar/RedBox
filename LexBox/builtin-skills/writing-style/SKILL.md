# Writing Style

用于当前空间内所有写作相关任务的系统内置风格技能。

这份技能不是临时模板，也不是过渡方案，而是当前空间长期维护、持续完善、不会被弃用的唯一写作风格技能。

## 强制规则

- 只要任务涉及写作、改写、扩写、润色、复盘、风格沉淀或作者表达方式调整，就必须激活本技能。
- 不要为同一空间继续创建平行的写作风格技能。
- 新的稳定观察、用户反馈、成稿复盘和风格修正，应优先回写本技能。
- 如果当前任务不是写作，不要让本技能主导非写作任务。

## 适用任务

- 写文章
- 改写已有稿件
- 扩写提纲或素材
- 润色成更像真人写的表达
- 复盘上一次成稿
- 沉淀个人风格规则

## 基础目标

- 像活人说话，不像报告或客服话术。
- 先讲具体场景、细节和感受，再讲抽象判断。
- 可以有观点，但不要装成无懈可击的上帝视角。
- 能承认不确定，就不要硬装确定。
- 能用真实经历、真实观察、真实处境支撑，就不要用空泛总结替代。
- 知识和观点要像聊着聊着自然掏出来，而不是突然切成教科书腔。
- 默认把读者当成一个具体的人去交流，不是对着一个模糊的大众群体喊话。

## 素材理解与选题判断

先吃透素材，再判断这件事值不值得写、适合怎么写。

默认用 HKR 做快速质检：

- H，是否足够有趣，能不能让人想继续读
- K，是否真的有信息量，而不是重复常识
- R，是否有情绪、处境或共鸣点

如果三项里只占一项，默认不要直接开写，先调整角度。
如果素材只有主题，没有细节、观点、经历或具体切口，先继续追问，不要硬写成空文。

## 常见文章原型

正式写之前，先判断更接近哪一类。

- 调查实验型，核心是我替你去做了一件事，再把过程和发现写出来
- 产品体验型，核心是我真的用了，再带你一起看
- 现象解读型，核心是一个具体现象背后到底在说明什么
- 工具分享型，核心是我发现了一个有用的东西，并且为什么值得你试
- 方法论型，核心是我把自己验证过的一套经验讲清楚，并把行动建议交给你

如果是方法论型，默认要求每个部分都能落到具体动作，而不是只输出正确废话。

## AI 协作边界

AI 默认适合做这些事：

- 梳理结构和段落顺序
- 找证据、背景、反例和类比
- 按已经确定的角度扩写
- 辅助做改写、压缩、润色和复盘

AI 默认不该擅自做这些事：

- 编造第一手经历
- 伪造真实情绪和现场细节
- 凭空生成作者根本没表达过的核心世界观
- 用假场景冒充真实案例

如果缺少真实信息，就坦白缺口，并向用户追问或降级表达。

## 结构习惯

- 开头默认从一个具体场景、当下事件、反常识事实、问题或好奇心切入，不从宏大空话起手
- 中段默认按 场景或现象 → 解释 → 进一步追问 → 观点或发现 推进
- 转场默认用口语化过渡、提问、回扣主线，而不是生硬的小标题拼接
- 收尾默认回到开头、留下余味、给出行动建议，或者把具体问题轻轻抬到更大的意义层
- 需要升华时，从具体经验自然抬高，不要突然论文式拔高

## 语言与节奏

- 默认长短句混用，避免整篇都像机器切出来的匀速句子
- 默认短段优先，重点句允许独立成段
- 默认允许适度口语和停顿感，但不要为了口语而刻意装腔
- 默认允许少量问句、感叹、留白和停顿，但要克制
- 讲观点时，先理解对立面的处境，再切入自己的判断
- 如果中途跑远了，要有一句把主线扣回来的话

## 绝对禁区

这些是最容易暴露AI味的地方，必须绝对避免：

套话：禁用"首先...其次...最后"、"综上所述"、"值得注意的是"、"不难发现"、"让我们来看看"、"接下来让我们"
过度结构化：不用bullet point罗列观点，不大量加粗。卡兹克的文章绝大多数时候没有小标题，从头到尾一口气顺下来，靠节奏和转场自然推进。唯一的例外是"N条心得/方法"这种本身就是独立条目的方法论文章，可以用数字编号（1、2、3），但也不是正式的markdown标题，就是文中的数字。如果不是这种分条目的结构，就不要加小标题，用口语化的转场句（"说到这个""回到xxx这块"）来衔接板块
标点禁令：
不使用冒号"："，用逗号代替
不使用破折号"——"
不使用任何双引号（""和""都不用），需要引用或强调时用「」或者直接不加引号
高频踩雷词，绝对禁用：
"说白了" ← AI特别爱用，一出现立刻暴露
"意味着什么？" ← AI标志性句式
"这意味着" ← 同上，换成更口语的表达
"本质上" ← 太学术
"换句话说" ← 太书面
"不可否认" ← 套话
假设性例子："比如有一次..."这种编造场景是大忌。要用"就像我今天正在搞的xxx"这种正在发生的真实细节。如果你没有真实细节，就别硬编，不如写"我自己还没试过，但想想就觉得xxx"
空泛工具名：不说"AI工具"、"某个模型"，要说具体名字，比如Claude Code、Codex、Seedance 2.0、Deepresearch、Clawbot
教科书开头：禁止"在当今AI快速发展的时代"、"随着技术的不断进步"这类空话开头。永远从一个具体的、当下的事件或场景切入

## 默认禁用词替换参考

- 说白了 → 坦率地讲、其实就是、直接点说
- 意味着什么 / 这意味着 → 那结果会怎样、所以呢、那最后会发生什么
- 本质上 → 说到底、其实、往深一层看
- 换句话说 → 你想想看、也就是说、换个更直白的讲法
- 不可否认 → 直接删掉，换成更具体的正面陈述
- 首先其次最后 → 改成自然转场，不强行凑报告顺序

## 写作自检

- 有没有具体场景、具体人、具体动作、具体问题
- 有没有只有表层口吻像人，底层判断却很空
- 有没有为了顺滑而写得太工整，工整到不像真人
- 有没有踩到禁区、空话、套话、假经历或假情绪
- 如果读者只记住一句话，会不会是具体而有判断力的那一句

## 迭代规则

- 每次成稿后的有效反馈、删改理由、用户明确喜欢或不喜欢的表达，都应优先写回本技能
- 如果新观察只是一次性噪音，先记为暂定，不要直接升级成长期规则
- 如果某条旧规则持续被真实样本打脸，就降级或删除
- 当前空间只维护这一份写作风格技能，不再另建平行版本
