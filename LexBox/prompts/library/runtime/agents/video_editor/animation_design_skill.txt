## 动画设计技能：Remotion 动画图层设计流程

你不是直接“猜一个动画结果”，而是必须按以下流程设计动画图层。

### 第一阶段：先判断动画主体来源
在开始设计动画前，先判断动画主体属于哪一类：

1. 手绘/图形对象
- 适用于几何图形、抽象形状、苹果、球、卡片、边框、背景块等
- 优先用 `entities[].type = "shape"`

2. 图标 / SVG
- 适用于 logo、图标、符号、品牌图形
- 优先用 `entities[].type = "svg"`

3. 文字
- 适用于标题、标语、字幕、强调语
- 优先用 `entities[].type = "text"`

4. 已有素材
- 适用于用户明确要求使用已有图片/视频素材
- 只有此时才用 `entities[].type = "image"` / `"video"` 或填写 scene binding

如果用户没有明确要求素材来源，默认优先使用：
- 文字对象
- shape 对象
- svg 对象

不要因为工程里已经有底层素材，就默认把动画绑定到底层素材。

### 第二阶段：决定对象表达，而不是先写说明文字
一旦动画主体来源确定，必须把主体转成对象表达：

- 不要把“苹果落地”写成说明性文字
- 要把它转成一个对象，例如：
  - `shape = "apple"`
  - `animations = [{ kind: "fall-bounce", ... }]`
- React / Remotion 层要围绕这个对象组织渲染与动画，而不是只生成 overlayTitle / overlayBody。

只有当用户明确要“屏幕上显示一句描述性文字”时，才把描述写成 `text` entity 或 overlay。
默认情况下，`overlayTitle`、`overlayBody`、`overlays` 都应保持为空，只保留动画对象本身。

### 第三阶段：先确定时间，再确定动画
动画设计时必须明确：

- 默认动画写入当前工程的主 scene，而不是按底层片段数量新建多个 scene。
- layer 什么时候开始：`fromMs` / `startFrame`
- 持续多久：`durationMs` / `durationInFrames`
- 如果有多个阶段，按多个 scene 或多个 entity animation 分段

默认原则：
- 动画不需要绑定片段
- 动画的核心是时间范围，而不是 clip 对齐

### 第四阶段：选择动画类型
根据对象和需求选择最接近的动画：

- 进入类：`fade-in`, `slide-in-left`, `slide-in-right`, `slide-up`, `slide-down`
- 强调类：`pop`
- 物体落地类：`fall-bounce`
- 漂浮类：`float`

如果用户描述的是更复杂动画，而宿主还不支持：
- 保留核心视觉意图
- 压缩成当前支持的对象 + 动画组合
- 在 `risks` 里明确指出能力降级点

### 第五阶段：输出约束
输出结果必须优先满足：

1. 能被宿主解析
2. 能进入 `M*` 动画轨
3. 能在 Motion / Preview 中直接预览
4. 不能依赖普通文字轨冒充动画

### 强制禁止
- 禁止通过 `text_add` / `subtitle_add` 往普通轨道补内容来伪装成动画
- 禁止无理由地把动画拆成“一个底层片段对应一个 scene”
- 禁止把对象动画退化成纯说明文字
- 禁止发明宿主未实现的新 DSL 或关键帧语法
