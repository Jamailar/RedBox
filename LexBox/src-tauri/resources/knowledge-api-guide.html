RedBox Knowledge Import Guide

知识库导入指南

RedBox 当前支持两种知识库导入方式：浏览器插件导入，以及本地 HTTP API 导入。 浏览器插件适合在网页里直接采集；API 适合你自己的脚本、自动化和第三方应用。下面这份页面只保留当前真实可用的插件能力和接口格式。

查看导入方式
方式 1

插件导入

在浏览器里看到内容时，直接导入到 RedBox 当前活动空间。

默认折叠
插件会自动探测本地 Knowledge API，无需手动填地址。当前支持小红书图文 / 视频 / 文章、抖音视频、YouTube 视频、网页链接 / 文章、选中文字，以及网页图片导入素材库。
1
打开 RedBox，进入 设置 → 工具，点击“一键准备插件”。
2
在 Chrome 或 Edge 打开扩展管理页，并开启开发者模式。
3
把 RedBox Browser Extension 文件夹拖进浏览器，或在“加载已解压的扩展程序”里选择该文件夹。
4
安装完成后，可在插件弹窗或右键菜单里执行“保存当前页面 / 保存选中文字 / 保存图片到素材库”等动作，导入结果会进入当前活动空间。
小红书图文 / 视频 / 文章 抖音视频 YouTube 视频 网页链接 / 文章 选中文字 网页图片 → 素材库
方式 2

API 导入

用于脚本、自动化流程或第三方工具接入。

默认折叠
支持内容

知识条目、图片素材。当前没有独立文档源接口，也没有混合批量导入接口。

服务入口

浏览器插件会自动探测；你自己接 API 时直接用这个地址。

http://127.0.0.1:31937/api/knowledge
应用保持运行即可使用；所有写入都会进入当前活动空间。

接口清单

可直接复制请求示例。

可用类型

`POST /entries` 当前建议使用这些 `kind`。

Source 字段

插件会自动补齐；自建调用至少建议传 `sourceLink / sourceUrl` 和 `externalId`。

source.appId / source.pluginId

来源标记。插件当前会自动附带；自建调用可选。

source.externalId

外部内容 ID。建议用于稳定去重和更新。

source.sourceDomain

来源域名。可不传，服务端可从链接推导。

source.sourceLink / source.sourceUrl

来源链接。当前优先消费 `sourceLink`，`sourceUrl` 保留兼容。

source.capturedAt

采集时间，可选；插件当前会自动写入。

响应说明

当前三个接口最常见的返回字段。

success

是否成功。

duplicate / updated

`/entries` 是否命中已有内容、是否更新了现有条目。

entryId / imported

`/entries` 返回 `entryId`；`/media-assets` 返回导入数量。

counts / items

`/health` 返回当前数量；`/media-assets` 返回导入后的素材列表。
