RedBox Knowledge Import Guide

知识库导入指南

RedBox 现在支持两种知识库导入方式：浏览器插件导入，以及自建 APP / 第三方 APP 的 API 导入。 这里先讲怎么选方式，再展开当前最新版接口格式、限制和请求示例。

查看导入方式
方式 1

插件导入

在浏览器里看到内容时，直接导入到 RedBox。

默认折叠
打开插件设置页
先在 RedBox 的 设置 → 工具 里点击“一键准备插件”。
1
打开 RedBox，进入 设置 → 工具，点击“一键准备插件”。
2
在 Chrome 或 Edge 打开扩展管理页，并开启开发者模式。
3
点击“加载已解压的扩展程序”，选择 RedBox 准备好的插件目录。
4
回到网页后，直接使用插件采集内容，导入结果会进入当前活动空间。
方式 2

API 导入

用于脚本、自动化流程或第三方工具接入。

默认折叠
支持内容

知识条目、文档源、图片素材、批量导入。

服务入口

复制地址后即可开始接入。

http://127.0.0.1:31937/api/knowledge
应用保持运行即可使用，导入内容会进入当前空间。

当前限制

提交前先看这三个限制。

Body Limit 16 MB
Batch Limit 64 Items
Entry Routes 5 个接口
更新时间 2026-04-19

接口清单

可直接复制请求示例。

可用类型

`POST /entries` 支持这些 `kind`。

Source 字段

常用来源字段说明。

source.appId / source.pluginId

来源标记。

source.externalId

外部内容 ID。

source.sourceDomain

来源域名。

source.sourceLink / source.sourceUrl

来源链接。

source.capturedAt

采集时间，可选。

响应说明

常见返回字段。

success

是否成功。

duplicate / updated

是否命中已有内容。

requestedActions

回显请求里的处理选项。

assets / source / results

不同接口返回的主要结果数据。
