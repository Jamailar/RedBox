主题目录的真实结构

当前仓库里，richpost 主题的主编辑目标是工作区下的 themes/<theme-id>/。主题目录索引可能同时存在兼容文件，但真正应该修改的是下面这些文件。

themes/ index.json # 当前主题目录索引 richpost-theme-template.md # 主题规则说明，给 AI 和开发者读 richpost-themes.json # 兼容/工作区级快照，可能存在 <theme-id>/ <theme-id>.json # 当前主题主配置 layout.tokens.json # 主题 token 真相层 page-plan.json # 当前主题分页方案 masters/ cover.master.html # 首页母版 body.master.html # 内容页母版 ending.master.html # 尾页母版 assets/ # 背景图和本地素材

每个文件负责什么

<theme-id>.json

保存主题记录本身，包括 coverFrame、bodyFrame、endingFrame，以及三种背景图路径和基础视觉字段。

layout.tokens.json

保存 CSS 变量层，是颜色、字号、行高、阴影、圆角、页面留白、字体等基础视觉的真相层。

masters/*.master.html

定义 cover、body、ending 三种母版的结构层，可以改背景层、卡片层、图片区、标题区、正文容器和装饰层。

page-plan.json

只有当 tokens 和母版不足以达成目标时才改。它控制页面使用哪张母版、zones 如何绑定、局部 styleOverrides 如何生效。

必须遵守的编辑顺序

1
先看规则文件 先阅读 themes/richpost-theme-template.md，再决定是改主题主配置、tokens、母版还是 page plan。
2
先改主题主配置和 tokens 颜色、字体、圆角、阴影、整体气质优先落在 <theme-id>.json 与 layout.tokens.json。
3
结构改动再动母版 涉及首页、内容页、尾页结构，去改 masters/cover.master.html、masters/body.master.html、masters/ending.master.html。
4
最后才改 page plan 只有在母版和 tokens 已经做不到的时候，才去改 page-plan.json 的 master、zones 或 styleOverrides。

关键实现细节

  • coverFrame、bodyFrame、endingFrame 是真实文字区域，字段为 x / y / w / h，使用 0 到 1 的相对坐标。
  • 文字区域还会同步映射为 layout.tokens.json 里的角色 CSS 变量，例如 --rb-frame-left、--rb-frame-top、--rb-frame-width、--rb-frame-height。
  • 背景图必须是本地资产，放进当前主题根目录下的 assets/，然后由 coverBackgroundPath、bodyBackgroundPath、endingBackgroundPath 引用。
  • 首页、内容页、尾页允许完全不同的构图，不要求三页统一模板。
  • 如果只是想让预览更像真实主题，不要只换颜色，要优先改模板文件本身。

绝对不要做的事

  • 不要改 content.md、content-map.json 等正文层文件。
  • 不要把正文文字直接硬编码进 masters/*.master.html。
  • 不要手改最终渲染结果文件，例如 pages/page-xxx.html。
  • 不要引入远程字体 URL、远程 CSS 或远程 JS。
  • 不要跨主题根目录乱改其他 themes/<other-theme-id>/ 文件。
  • 不要为了达成视觉效果就重写 page plan；先把主题主配置、tokens 和母版用满。
